    Внутрішні повідомлення

    Основні команди

    npm start - старт сервера Node в DEV env

    npm run dev - старт supervisor над Node в DEV env, для відслідковування змін у файлах і миттєвого перезапуску сервера

    npm run production - старт сервера Node в PROD env

    npm run test - запуск Unit тестів

    npm run docs - генерація документації для сервісу і API. Документація формується в директорії docs.

    Внутрішня структура програмного інтерфесу

    Точка входу в сервіс bin/www. Каталог bin/ потрібен в якості місця, де можна створити різні сценарії запуску сервісу. В файлі www.js підключаються необхідні модулі для запуску сервера, завантажуються файли конфігурації, середовища .env, Декларуються handlers для обробки помилок запитів.

    config/ - каталог з файлами і функціями конфігурації.

    config.json - основні налаштування сервісу. environment.js - функція - селектор для підключення файлу змінних середовища в залежності від конфігурації запуску (development, test, production) init-app.js - підготовка сервісу до роботи. (отримання даних про активних користувачів, налаштувань, запуск демона node-cron для виконання завдань по часу) mongoose.js - підключення і конфігурування драйвера MongoBD Mongoose

    models/ - каталог моделей, з методами для внесення змін в БД

    modules/ - модулі для комунікації з клієнтом по різним протоколам (websocket, web-push і т.д.)

    web-push.js - модуль для відправки повідомлень через web-push ws.js - модуль для выдправки повідомлень через Websocket

    public/ - каталог статичних файлів з клієнтським кодом та заглушками.

    routes/ - каталог маршрутів

    test/ - каталог тестів

    .env - файли середовища

    worker.js - файл клієнського воркера (необхідно, щоб знаходився в корні)

    Структура БД

    БД сервісу складається з двох колекцій: users, messages для активних користувачів і повідомлень відповідно.

    Структура колекції users

    {
    globalId: String,
    group: Array[String, String, ...],
    userOptions: {
        timezone: String,
        deliveryСhannels: Array[String, String, ...]
    },
    isOnline: Boonean,
    subscribe: Object {
        endpoint: String,
        expirationTime: null,
        keys: {
             p256dh: String,
             auth: String
        }
    }
}

    globalId - глобальний Id користувача в системі. Зберігаються всі користувачі в системі (необхідно для збереження підписок web push notifications)

    group - массив груп до яких належить користувач.

    userOptions - Налаштування користувача. timezone - Налаштування таймзони. deliveryСhannels - Сервер відправки повідомлень (наприклад: telegram, електронна пошта, браузер – в залежності від типу повідомлення) isOnline - Активність користувача в системі.

    subscribe - Об'єкт підписки користувача. endpoint - Кінцева точка, унікальний url для доставки сповіщення користувачу. keys - Необхідні для передачі payload з сервера (JSON, ArrayBuffer, BLOB)

    Структура колекції messages

    {
    serviceName: String,
    status: String,
    groups: Array [String, String, ...],
    users: Array [String, String, ...],
    addressee: String,
    text: String,
    dispatchTime: { type: GMTString, default: Date.now },
    sendingServer: [String, String, ...],
    isViewed: Boolean,
    createdDate: Date GMT String,
    actualityViewedDate: Date GMT String,
    actualityNotViewedDate: Date GMT String
}

    serviceName - Назва сервісу, що відправив повідомлення

    status - Статус важливості повідомлення

    groups - Групи користувачів, яким мають отримати повідомлення

    users - Список користувачі, що мають отримати повідомлення

    addresse - globalId отримувача повідомлення

    text - Текст повідомлення

    dispatchTime - Час доставлення повідомлення, якщо це передбачено. Формат Date GMT String

    sendingServer - Сервер відправки повідомлень

    isViewed - Повідомлення переглянуто чи ні

    createdDate - Час створення повідомлення

    actualityViewedDate - Час збереження прочитаного повідомлення в системі - встановлюється з налаштувань системи; Date GMT String

    actualityNotViewedDate - Час збереження НЕпрочитаного повідомлення - встановлюється з налаштувань системи. Date GMT String
